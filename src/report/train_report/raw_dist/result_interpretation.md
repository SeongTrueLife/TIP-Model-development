
# 모델별 Raw 점수 분포 분석 및 Node 0 튜닝 방향

**분석 일자:** 2026-02-18
**참조 그래프:** `src/report/train_report/raw_dist/` 폴더 내 이미지

## 1. 외관 모델 (Visual - Model 2)
*   **상태:** **가장 구분이 어렵습니다.**
    *   **유사(Similar) 평균:** 0.49
    *   **비유사(Dissimilar) 평균:** 0.39
    *   **분석:** 점수 차이가 0.1 밖에 나지 않고, 분포가 거의 겹쳐 있습니다. (Standard Deviation ~0.32)
    *   **튜닝 방향:** 
        *   유사 케이스의 하위 25%가 0.22점입니다. 
        *   비유사 케이스의 상위 25%가 0.67점에서 시작합니다.
        *   즉, **0.2 ~ 0.7 구간**은 회색지대(Gray Zone)로, 낮게 깔아주는 보수적인 맵핑이 필요합니다.

## 2. 호칭 모델 (Phonetic - Model 3)
*   **상태:** **가능성이 보입니다 (평균의 함정 주의).**
    *   **유사 평균:** 49점 vs **비유사 평균:** 35점 (차이가 작아 보임)
    *   **중앙값(Median) 비교:** **유사 69점** vs **비유사 13점**
    *   **분석:** 평균은 0점 데이터들 때문에 낮아져 있지만, 중앙값을 보면 **13점 vs 69점**으로 확실한 차이가 있습니다.
    *   **튜닝 방향:** 
        *   현재 점수가 너무 낮게 평가되고 있습니다.
        *   **40점~50점 구간**부터 점수를 확 올려줘도 됩니다. (비유사의 75%가 72점 이하이므로, 70점 이상은 고득점 처리)

## 3. 관념 모델 (Semantic - Model 4)
*   **상태:** **변별력이 좁은 구간에 몰려 있습니다.**
    *   **유사 평균:** 0.70
    *   **비유사 평균:** 0.62
    *   **분석:** 비유사 데이터도 기본적으로 0.6점 이상을 받고 있어, **False Positive(오탐)**의 주원인입니다.
    *   **튜닝 방향:** 
        *   **Stretch(늘리기) 전략**이 필요합니다.
        *   0.6점(비유사 평균) 근처는 점수를 **대폭 깎고(Suppress)**,
        *   0.75점 이상부터 점수를 부여하도록 **기준을 상향**해야 합니다.

---
**종합 제안:**
1.  **호칭(Phonetic):** 40점대부터 점수가 올라가도록 Anchor 조정 (현재 너무 짬)
2.  **관념(Semantic):** 0.6~0.7 구간 점수를 대폭 낮추고, 0.75 이상부터 인정하도록 조정 (현재 너무 후함)
3.  **외관(Visual):** 겹침이 심하므로, 보수적으로 설정 유지 (큰 기대 난망)
