
# Node 0 (Calibrator) 튜닝 보고서 - 호칭 모델 (Phonetic)

**작성일:** 2026-02-18
**대상 모델:** Model 3 (Phonetic)
**목표:** 0점(Error) 데이터를 배제하고, 유효한 호칭 유사도 점수에 대한 최적의 맵핑 전략 수립

## 1. 데이터 현황 비교 (Original vs Filtered)

### A. Original (0점 포함)
*   **왜곡:** 0점 데이터(발음 변환 실패)가 평균을 대폭 깎아먹어, 실제 모델의 성능을 과소평가하게 만듦.

### B. Filtered (0점 제외 - Valid Only)
*   **유사(Similar):** 평균 **80점** (중앙값 86점)
*   **비유사(Dissimilar):** 평균 **61점** (중앙값 70점)
*   **Easy Negative:** 평균 **11점** (중앙값 6점)

## 2. 튜닝 전략: "Filtered 데이터 기준 S-Curve"

> **질문:** 어떤 데이터를 기준으로 설계하는게 나을까? 
> **답변:** 무조건 **Filtered(0점 제외) 데이터** 기준이어야 합니다.

### 제안 앵커 포인트 (S-Curve)

| Raw Score (입력) | Target Score (출력) | 의도 |
| :--- | :--- | :--- |
| **0 ~ 50** | 0.1 | 노이즈 제거 (Easy Negative 등) |
| **60** | **0.3** | 비유사 평균(61점). 0.5 미만으로 낮춰 비유사 판정 유도. |
| **73** | **0.55** | **[변곡점]** 비유사 상위권과 유사 하위권의 경계. |
| **80** | **0.8** | 유사 평균(80점). 확실한 침해. |
| **100** | 1.0 | 동일 |

## 3. 시뮬레이션 결과 (Impact Analysis)

기존 코드의 앵커와 제안된 앵커를 비교 시뮬레이션한 결과입니다. (Valid Data 59건 기준)

### A. 유사(Similar) 케이스 놓침 (점수 < 0.5)
*   **현재:** 8건 놓침
*   **제안:** **8건 놓침 (변화 없음)**
*   **해석:** 제안된 튜닝이 유사 케이스를 더 놓치게 만들지 않습니다. 안전합니다.

### B. 비유사(Dissimilar) 케이스 오탐 (점수 >= 0.5)
*   **현재:** 11건 오탐
*   **제안:** **10건 오탐 (1건 개선)**
*   **해석:** 60~70점 구간의 점수를 미세하게 낮춤으로써, 오탐을 소폭 줄이는 효과가 있습니다.

## 4. 최종 제안
호칭 모델은 비유사 중앙값(70점)이 꽤 높기 때문에, **70점 초반까지 점수를 억제**하다가 **80점에서 급등**시키는 S-Curve 전략이 가장 유효합니다. 시뮬레이션 결과 리스크(Recall 하락) 없이 안정성(Precision)을 소폭 높일 수 있습니다.
