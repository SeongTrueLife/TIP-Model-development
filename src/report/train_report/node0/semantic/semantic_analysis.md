
# Node 0 (Calibrator) 튜닝 보고서 - 관념 모델 (Semantic)

**작성일:** 2026-02-18
**대상 모델:** Model 4 (Semantic)
**목표:** 유사/비유사 구간 겹침(Overlap)이 심한 상황에서 오탐(False Positive)을 최소화하는 튜닝

## 1. 데이터 분포 정밀 분석 (Percentile)

| 구분 | 판례-유사 (Similar) | 판례-비유사 (Dissimilar) | Easy Negative |
| :--- | :--- | :--- | :--- |
| **Median (중앙값)** | **0.71** | **0.64** | 0.44 |
| **10% ~ 90% 구간** | 0.58 ~ 0.81 | 0.46 ~ 0.74 | 0.35 ~ 0.52 |
| **Max (최대값)** | 0.84 | 0.77 | 0.55 |

**핵심 문제점:**
1.  **Severe Overlap:** 유사 하위 10%가 0.58인데, 비유사 상위 25%가 0.70을 넘습니다. 즉, 0.6~0.7 구간은 아예 구분이 불가능한 **혼돈의 영역**입니다.
2.  **High Baseline:** 비유사 데이터도 기본 0.6점 이상을 깔고 들어갑니다. (False Positive 위험 매우 높음)

## 2. 튜닝 전략: "Suppress & Stretch (누르고 늘리기)"

이 모델은 변별력이 좁은 구간(0.6~0.8)에 몰려 있으므로, 이 구간을임의로 찢어발겨야(Stretch) 합니다.

1.  **Suppress (누르기):** 
    *   비유사 중앙값이 0.64입니다. 따라서 **0.65점까지는 점수를 주면 안 됩니다.** (0.5 미만 처리)
    *   기존에는 0.6만 넘어도 0.8점(높음)을 줬는데, 이게 오탐의 주원인입니다.

2.  **Stretch (늘리기):**
    *   유사 중앙값이 0.71입니다. 따라서 **0.7점부터 점수가 급격히 올라가도록** 해야 합니다.

### 제안 앵커 포인트

| Raw Score (입력) | Target Score (출력) | 의도 |
| :--- | :--- | :--- |
| **0 ~ 50** | 0.1 | 하위권 노이즈 제거 |
| **0.6** | **0.2** | **[제재]** 비유사 절반 가량이 여기에 속함. 0.5 미만으로 강력 억제. |
| **0.65** | **0.4** | **[경계 1]** 비유사 중앙값(0.64) 통과 지점. 아직은 비유사로 봄. |
| **0.7** | **0.65** | **[경계 2]** 유사 중앙값(0.71) 근처. 가능성 열어둠. |
| **0.75** | **0.85** | **[확신]** 비유사 상위 90%가 0.73임. 0.75 넘으면 찐유사. |
| **1.0** | 1.0 | 동일 |

## 3. 기대 효과
*   **오탐 감소:** 기존에 0.6점(비유사) 맞고 0.8점(유사)으로 둔갑하던 애들을 0.2점(비유사)으로 제자리로 돌려보냅니다.
*   **변별력 강화:** 0.6~0.75라는 좁은 구간(0.15 차이)을 0.2~0.85(0.65 차이)로 확장하여, 미세한 점수 차이가 결과에 크게 반영되도록 합니다.
