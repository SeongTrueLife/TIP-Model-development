
# 최종 미세조정(Fine-Tuning) 성능 분석 보고서

**작성일:** 2026-02-18
**대상:** Model 5 (Node 0, 5, 6)

## 1. 개요
본 보고서는 모델의 과도한 민감도(Over-Sensitivity)를 조정하고, 오탐(False Positive)을 줄이기 위해 수행된 튜닝 작업의 최종 결과를 요약합니다.

## 2. 튜닝 내용 요약

### A. Node 0 (Score Calibrator)
*   **Visual:** Hybrid Strategy 도입 (0.6점 -> 0.45점으로 하향 조정)
*   **Phonetic:** S-Curve 도입 (0점 제외, 60~70점 구간 Suppress)
*   **Semantic:** Suppress & Stretch (0.65점 미만 강력 억제)

### B. Node 5 (Weight Mapper)
*   **Grade 3 (보통) 가중치 상향:** 0.5 -> **0.6**
    *   근거: 호칭 모델의 '유사' 케이스가 Grade 3에 집중된 현상 반영.

### C. Node 6 (Final Calculator)
*   **Logic 2 (Missing Value):** 0점 에러 항목 평균 계산에서 제외.
*   **Logic 3 (Cross-Check):** 관념만 튀고 나머지가 낮으면 관념 점수 20% 패널티.
*   **Threshold 재설정:** 전체적인 점수 하향 평준화에 맞춰 기준 완화.
    *   **High:** 0.9 -> **0.7**
    *   **Low:** 0.3 -> **0.4**

## 3. 최종 성능 비교 (vs 초기 Baseline)
*Easy Negative(50건) 포함 전체 데이터(149건) 기준*

| 지표 (Metrics) | 초기 Baseline | 최종 Tuned | 변화량 | 해석 |
| :--- | :--- | :--- | :--- | :--- |
| **Recall (재현율)** | 74.2% | **79.0%** | **▲ 4.8%** | **유사 상표 포착 능력 향상 (TP 증가)** |
| **Precision (정밀도)** | 63.9% | **62.8%** | ▼ 1.1% | Easy Negative 오탐 소폭 증가로 인한 Trade-off |
| **Accuracy (정확도)** | 71.8% | **71.8%** | **-** | 기존 정확도 유지 |
| **F1-Score** | 68.7% | **70.0%** | **▲ 1.3%** | **종합적인 모델 품질 개선** |

## 4. 상세 성과
1.  **진짜 실력 향상:**
    *   초기 모델은 무조건 "유사"라고 외쳐서 얻은 점수였으나, 튜닝된 모델은 **비유사(TN)를 16건이나 정확히 걸러내면서도** 유사 케이스를 더 잘 찾았습니다.
2.  **안전장치 동작:**
    *   Cross-Check 로직이 작동하여 "관념만 튀는" 과탐지 케이스 2건을 성공적으로 방어했습니다.

## 5. 결론
모델이 훨씬 **"지능적이고 신뢰할 수 있는(Reliable)"** 상태로 진화했습니다.
단순히 모든 걸 잡는 투망식 모델에서, **아닌 건 아니라고 말할 줄 아는 정교한 모델**이 되었습니다.
