
# Node 5 (Weight Mapper) 튜닝 보고서

**작성일:** 2026-02-18
**대상:** Grade 3 (보통) 가중치 상향 조정 (0.5 -> 0.6)

## 1. 개요 및 목적
LLM이 판단한 식별력 등급(Grade 1~5)을 수치형 가중치(Weight)로 변환하는 과정에서, "보통(Grade 3)" 등급의 중요도를 상향하여 전반적인 Recall(재현율)을 개선하고자 함.

## 2. 데이터 분포 분석 (Grade 3 집중 탐구)

Grade 3를 받은 데이터가 어느 카테고리에 몰려있는지 확인한 결과입니다.

| 모델 | 유사(Similar) 케이스 | 비유사(Dissimilar) 케이스 | Grade 3의 성격 |
| :--- | :--- | :--- | :--- |
| **Visual** | **13건** | 7건 | 유사 케이스 비중이 약 65%로 높음. |
| **Phonetic** | **20건** | 8건 | **유사 케이스 비중이 압도적(71%).** |
| **Semantic** | 8건 | 7건 | 반반 (중립적). |

**해석:**
*   특히 **호칭 모델**에서 LLM이 "유사하지만 확실하진 않다"고 느낄 때 Grade 3를 주는 경향이 강함.
*   비유사 케이스가 Grade 3를 받는 경우는 상대적으로 적음.

## 3. 시뮬레이션 및 결론

**가설:** Grade 3 가중치를 0.5 -> 0.6으로 올리면?
*   **유사 케이스 영향:** 총 41건(13+20+8)의 유사 케이스가 점수 상승 효과를 봄. 특히 호칭 유사 건들의 회복세가 뚜렷할 것임.
*   **비유사 오탐 위험:** 총 22건(7+8+7)의 오탐 점수도 오르지만, 유사 케이스 상승폭이 2배 더 큼.
*   **Node 0 튜닝과의 시너지:** 이미 Node 0에서 비유사 점수는 억제(Suppress)해놨으므로, 가중치가 조금 올라도 최종 점수는 크게 튀지 않을 것임.

**최종 제안:**
*   **Grade 3 -> 0.6 상향:** **적극 찬성.** (Recall 증대 효과 확실)
*   **Grade 5 (1.0), Grade 4 (0.8):** 현행 유지. (이미 충분함)
*   **Grade 2 (0.2), Grade 1 (0.05):** 현행 유지. (노이즈 필터 역할 충실)

## 4. 변경 예정 코드 (src/model5/nodes/node_5_weight_mapper.py)
```python
GRADE_WEIGHT_MAP: Dict[int, float] = {
    5: 1.0,  # Exclusive
    4: 0.8,  # Strong
    3: 0.6,  # Moderate [변경: 0.5 -> 0.6] 점수 보존율 높임
    2: 0.2,  # Weak
    1: 0.05  # None
}
```
