# 모델3(호칭 유사도 모델) 수정 사항 공유 및 적용 가이드

안녕하세요 재현님,
모델3(호칭 유사도 모델) 관련하여 **치명적인 버그(Cross-Contamination) 수정**을 완료했습니다.
통합 프로젝트에 아래 내용을 반영 부탁드립니다.

## 📌 1. 핵심 변경 사항
- **수정된 파일:** `code/converter.py` (단 1개 파일)
- **변경 내용:** 
  - 기존에는 두 상표(A, B)를 한 번에 API로 호출했으나, GPT가 두 상표의 발음을 섞어서 반환하는 환각 현상(Hallucination)이 발견되었습니다.
  - 이를 방지하기 위해 상표를 하나씩 독립적으로 변환(`convert_single`)하도록 로직을 수정했습니다.
  - **결과:** API 호출 횟수는 2배가 되지만, 데이터 무결성과 정확도가 100% 보장됩니다.

## 🛠️ 2. 적용 방법
이 폴더(`model3/model3_final`)에 있는 **`code/converter.py`** 파일을 복사하여, 현재 통합 진행 중인 프로젝트의 해당 경로에 **덮어쓰기(Overwrite)** 해주세요.

> **참고:** `main.py`, `scorer.py`, `evaluate.py` 등 다른 파일들은 기존 버전과 동일하므로 수정할 필요 없습니다.

## ⚠️ 3. 특이 사항 (0점 이슈 관련)
테스트 결과(`baseline_result_train_first_train.csv`)에서 일부 데이터의 유사도 점수가 0점으로 나오는 현상이 발견되었습니다.
- **원인:** 모델 코드가 아닌, **입력 데이터(CSV)의 인코딩 깨짐(한글 깨짐)** 때문입니다.
- **조치:** 모델 코드는 정상 작동 중이므로 수정 불필요하며, 향후 원본 데이터를 UTF-8로 다시 추출하여 입력하면 해결될 문제입니다.
