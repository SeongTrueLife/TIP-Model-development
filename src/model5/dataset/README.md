# Model 5 미세조정 및 통합 테스트 데이터셋 구축 가이드

이 문서는 **Model 2, 3, 4, 5의 통합 성능 검증** 및 **Model 5의 핵심 노드(Node 0, 5, 6) 파라미터 최적화(Fine-tuning)**를 위한 데이터셋 구축 방법을 설명합니다.

---

## 1. 데이터셋 개요

*   **목적**:
    1.  개별 모델(Visual, Phonetic, Semantic)의 점수 분포를 파악하여 **정규화(Calibration) 임계값 보정**.
    2.  실제 판례에서 중요하게 다뤄진 요소(외관/호칭/관념)를 분석하여 **가중치(Weight) 및 요부 관찰(Dominant Rule) 로직 미세조정**.
    3.  통합 파이프라인(Pipeline)의 최종 **침해 여부 판단 정확도 검증**.
*   **파일 형식**: Excel (`.xlsx`)
*   **이미지 저장**: 데이터셋 엑셀 파일과 동일한 위치(또는 하위 `images/` 폴더)에 이미지 파일 저장 권장.

---

데이터 수집 시 아래의 컬럼 헤더(Header)를 사용하여 엑셀 파일을 작성해 주십시오.

| 그룹 | 컬럼명 (Header) | 필수 여부 | 설명 및 입력 예시 |
| :--- | :--- | :--- | :--- |
| **Meta** | `case_id` | 필수 | 판례번호 또는 고유 ID (예: `2023허1234`, `data_001`) |
| | `split` | 필수 | `train` (모델 튜닝용) 또는 `test` (최종 검증용) 구분 |
| **Input (Target)** | `target_text` | 필수 | 출원상표(검사 대상)의 문자열 (예: `STARBUCKS`) |
| | `target_img` | 필수 | 출원상표 이미지 파일명 (예: `target_001.jpg`) |
| | `target_prod` | 필수 | 지정상품 (예: `커피`, `의류`) |

## 3. 이 데이터가 왜 필요한가요? (Why?)

모델을 정확하게 튜닝하기 위해서는 단순한 O/X(유사/비유사)를 넘어선 **'이유(Reasoning)'** 데이터가 필요합니다.

### 🎯 `main_factor` (주요 판단 요인)
*   **용도:** **Node 0 (Calibrator) 및 Node 6 (Calculator) 튜닝**
*   **이유:** 
    *   판결에서 **"외관은 다르나 호칭이 유사하여 침해다"**라고 했다면, 이 데이터는 **Model 3(호칭) 모델의 튜닝**에는 쓰여야 하지만, **Model 2(외관) 모델의 튜닝에는 쓰이면 안 됩니다.** (외관이 다르다는 데이터를 외관 모델에 '유사'라고 학습시키면 모델이 망가짐)
    *   따라서 `main_factor`에 '외관'이 적힌 데이터만 골라서 Model 2를 튜닝하고, '호칭'이 적힌 데이터만 골라서 Model 3를 튜닝합니다.

### 📊 `sim_level` (유사도 강도)
*   **용도:** **Node 0 (Calibrator) 목표 점수 설정 (Soft Labeling)**
*   **이유:**
    *   똑같이 '유사' 판결이라도, **"누가 봐도 똑같은(High)"** 경우와 **"애매하지만 법리상 유사한(Low)"** 경우가 있습니다.
    *   Model 0이 High 케이스는 **0.9점 이상**으로, Low 케이스는 **0.6점 근처**로 매핑하도록 유도하여 모델의 섬세함을 높입니다.

## 4. 데이터 작성 팁 (변리사 수험 경험 활용)

### `main_factor` 작성 가이드
판결문을 읽고 가장 쟁점이 되었거나 판사가 "유사하다"고 콕 집어 말한 부분을 적습니다.
*   **외관:** 도형의 생김새, 로고 디자인이 쟁점인 경우.
*   **호칭:** 발음(소리)이 비슷해서 쟁점인 경우. (대부분의 케이스)
*   **관념:** 뜻(의미)이 비슷해서 쟁점인 경우. (예: `CROWN` vs `왕관`)
*   **전체:** 딱히 하나를 꼽기 어렵고 전체적인 인상이 비슷한 경우.

### `sim_level` 판단 가이드 (간소화 기준)
너무 고민하지 마시고, 직관적으로 판단하십시오.

1.  **High (동일 수준)**
    *   **기준:** 오타 수준의 차이, 혹은 글씨체/색상만 살짝 바꾼 경우. 누가 봐도 "이건 베꼈네" 싶은 경우.
    *   **작성:** 자신 있게 `High` 입력.

2.  **Medium (일반 유사)**
    *   **기준:** 글자 수가 다르거나 도형이 추가되었지만, 핵심 요부(Dominant Part)가 같아서 헷갈리는 경우. 대부분의 침해 사례.
    *   **작성:** `Medium` 입력 (또는 비워두면 기본값으로 처리됨).

3.  **Low (경계선)**
    *   **기준:** 1심/2심 판결이 갈렸거나, 판결문 논리가 복잡하고 "전체적으로 관찰하면 비유사하나..." 같은 단서가 붙는 경우.
    *   **작성:** 판단이 어렵다면 비워두세요.
