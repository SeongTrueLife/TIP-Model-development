# [모델 5] 법리 기반 동적 가중치 앙상블 모델 개발 명세서

(Technical Specification: Legal-based Dynamic Weighting Ensemble)

## 1. 개요 (Overview)

### 1.1 목적

모델 2(외관), 3(호칭), 4(관념)에서 산출된 기초 유사도 점수를 단순히 수학적으로 평균 내는 것이 아니라, **대한민국 상표법의 핵심 법리(식별력 유무, 요부 관찰)**를 적용하여 "어떤 유사도가 더 중요한지(가중치)"를 동적으로 판단하는 최종 의사결정 시스템이다.

### 1.2 핵심 원리

- **식별력(Distinctiveness)에 따른 가중치 조절**: 식별력이 없는 부분(예: 보통명칭, 관용명칭, 성질표시, 현저한 지리적 명칭, 간단하고 흔한 표장, 흔한 성 또는 명칭, 기타 식별력 없는 표장)의 유사도는 가중치를 '0'에 가깝게 낮추고, 식별력이 강한 부분의 유사도에 가중치를 높인다.
- **요부(Dominant Part) 중심 판단**: 상표 전체에서 지배적인 인상을 주는 요소(요부)가 무엇인지 파악하여 해당 영역의 가중치를 극대화한다.
- **하이브리드 관찰 로직**: 침해 판단 시 '요부 관찰(치명적 유사)'과 '전체 관찰(전반적 유사)'을 분리하여 적용함으로써, 단순 평균 계산으로 인한 위험도 희석(Dilution) 문제를 방지한다.

## 2. 입력 데이터 명세 (Input Specifications)

시스템은 Metadata와 Base Scores 두 가지 입력을 받는다.

### 2.1 메타 데이터 (Metadata)

**Target (등록 상표):**

- target_img: 이미지 파일 (경로 또는 바이너리)
- target_text: 상표 텍스트 (DB 데이터)
- target_product: 지정상품명 (예: "제25류 의류, 등산화")
- target_sem_desc: 관념 묘사 텍스트 (모델 4의 중간 산출물)

**Candidate (침해 의심 상표):**

- cand_img, cand_text, cand_product(사용상품명), cand_sem_desc (Target과 동일 구조) -> 제거. (mvp 단계에서는 모델5에서 인풋 데이터로 안쓸거임.)

### 2.2 기초 유사도 점수 (Base Scores)

- $S_{vis}$ (Visual Score): 모델 2 출력값 (Range: 0.0 ~ 1.0)
- $S_{pho}$ (Phonetic Score): 모델 3 출력값 (Range: 0 ~ 100) (※ 연산 시 0.0~1.0으로 정규화 필수)
- $S_{sem}$ (Semantic Score): 모델 4 출력값 (Range: 0.0 ~ 1.0)

## 3. 상세 워크플로우 (Detailed Workflow)

전체 파이프라인은 **[Phase 1: 시각적 심층 분석] → [Phase 2: 법리 판단 (RAG & Logic)] → [Phase 3: 가중치 산출] → [Phase 4: 최종 연산]**으로 구성된다.

### Phase 1: 시각적 심층 분석 (Visual Analysis)

각 영역을 전담하는 LLM Agent들이 상표를 분석하여 정성적(Qualitative) 데이터를 생성한다.

#### Agent A: 외관 묘사관 (Visual Describer)

**목적**: 이미지의 객관적 형태를 텍스트화하여, 후속 Agent B에게 판단의 근거(Reference)를 제공하고 최종 사용자에게 설명(XAI)할 자료를 만든다.

**모델**: GPT-5.1 (Vision Capability)

**입력:**

- target_img: 등록상표 이미지 (필수)

**Role (역할 및 지침):**

- "주관적 해석 금지, 객관적 묘사 집중"
- 이미지의 색상, 도형의 형태(원, 사각형 등), 텍스트의 배치, 구성요소의 크기 비율 등을 '보이는 대로' 서술한다.
- 식별력 유무(흔하다, 독특하다 등)는 이 단계에서 절대 판단하지 않는다. 오직 형태적 팩트만 추출한다.

**System Prompt 예시:**
"당신은 기하학적 형태 분석가입니다. 상표 이미지를 보고 다음 요소를 객관적으로 묘사하십시오. 의미(예: '왕관', '사자')는 해석하지 말고, 형태(예: '뾰족한 윗부분을 가진 노란색 원형') 위주로 서술하십시오."

1. 기하학적 특징 및 도안의 복잡도
2. 텍스트와 도형의 배치 구도
3. 전체 이미지 대비 텍스트/도형의 면적 비중

**Output:**

- vis_desc_text (String): 예) "검은색 테두리의 원형 안에 고딕체로 'ABC'가 기재되어 있으며, 하단에는 붉은색 리본 도형이 배치됨."

### Phase 2: 법리적 식별력 판단 (Legal Judgment with RAG)

이 단계에서는 30만 건의 거절결정 데이터를 기반으로 구축된 고도화된 RAG 시스템을 활용하여, Agent B가 최종적인 법리 판단을 수행한다.

#### 2-1. RAG 시스템 파이프라인 (RAG Pipeline)

**목표**: 30만 건의 원천 데이터를 최소한의 비용으로 정제하여 "검색 가능한 지식"으로 변환한다.

##### 데이터 전처리 (Data Preprocessing): 소극적 클렌징 (Safe Slicing & 원본 보존)

**대상**: 특허청 거절결정서 (rejectionContentDetail 컬럼). (※ 다른 거절이유 제외, 식별력 관련 데이터 집중->데이터가 식별력을 원인으로 한 거절 걸정된 경우로 따로 모와져 있음.)

**텍스트 클렌징 전략**: 완벽한 정형화를 포기하고, 명백한 행정적 노이즈만 제거하여 원본의 맥락을 보존한다.

- Head: "참고 : 상표견본이미지..." 등 판례와 무관한 안내 문구 삭제.
- Tail: "지정상품 - 전부. 끝." 등 단순 나열식 문구 삭제.
- Body: "본원상표는...", "상표법 제6조..." 등 본문 내용은 절대 건드리지 않고 유지한다.

**구법(제6조) 처리**: 데이터를 인위적으로 수정하지 않는다. 대신 Agent B의 시스템 프롬프트에 "구법(제6조)을 신법(제33조) 논리로 해석하라"는 지침을 추가하여 해결한다.

##### 메타데이터 태깅 (Auto-Tagging)

LLM 대신 가벼운 키워드/Regex(정규식)를 활용하여 비용을 절감하고 속도를 확보한다.

**Reason Tags (거절사유)**: - 아래 정규식 매칭 키워드들은 조정 필요-

- TAG\_성질표시: 성질, 효능, 용도, 원재료, 품질 등
- TAG\_보통명칭: 보통명칭, 관용
- TAG\_현저한지리: 지리적, 현저한, 지도
- TAG\_간단흔한표장: 간단하고 흔한, 1자, 2자 등
- TAG\_흔한성명칭: 흔한 성 또는 명칭
- TAG\_기타: 위 키워드 미포함 시

**Product Tags (상품분류)**: Nice 분류 100대 키워드 매칭 (예: CLS*식품, CLS*전자, CLS\_의류 등 대분류 태그 부착).

##### 데이터베이스 구축:

- **Vector DB**: ChromaDB (Local or Server)
- **Embedding Model**: bge-m3 (Korean Optimized)
- **Data Structure:**
  - Document: "상표명 : 거절이유요약"
  - Metadata: { "reason*tag": ["TAG*성질표시"], "product*tag": "CLS*식품" }

#### 2-2. 검색 전략 (Retrieval Workflow): 논리적 분해 검색 (Decomposition Search)

단순 텍스트 매칭이 아닌, "법리적 논리"와 "상품 맥락"이 일치하는 심사 거절이유를 찾아낸다.

##### Step 1: 쿼리 및 필터 생성 (Query Generation)

**Actor**: LLM (gpt-4o-mini 권장)

**Input**: Target 상표명, 상표 외관에 관한 묘사 텍스트(Agent a 에서 나온), 상표 관념에 관한 묘사 텍스트(Agent b 에서 나온), 지정상품

**Prompt**: "너는 검색 전문가야. 상표 맛있는우유(식품)의 식별력을 판단하기 위해 판례를 검색하려 해. 다음 3가지 관점의 검색어(Query)를 JSON으로 뽑아줘.">

- 1. Literal: 상표명 그 자체나 분할된 단어.>
- 2. Semantic: 상표의 구조적/추상적 특징 설명문.>
- 3. Category: 지정상품과 관련된 대표 키워드.

**Output (JSON Example):**

```json
{
  "queries": [
    "맛있는 우유 식별력", // Literal: 상표명 자체/포함 단어
    "형용사 수식어와 보통명칭 결합 표장", // Semantic Pattern: 추상적 구조
    "유제품 성질표시 거절 사례" // Category: 지정상품군 내 기준
  ],
  "filter_tags": ["TAG_성질표시", "CLS_식품"]
}
```

##### Step 2: 하이브리드 검색 (Hybrid Search)

- **Querying**: 생성된 3개 쿼리로 각각 벡터 검색 수행 (ChromaDB.query).
- **Filtering**: where={"$and": [{"reason_tag": ...}, {"product_tag": ...}]} 조건 적용.
- **Reranking**: 검색된 결과(총 9~15개) 중 유사도 점수 상위 3~5개를 선별하여 Context로 활용.

#### 2-3. Agent B: 통합 법리 심사관 (Legal Judge)

**목적**: 시각 정보(Image), 텍스트 정보(Desc), 법리 정보(RAG)를 종합하여 식별력의 강약과 요부(Dominant Part)를 최종 판단한다.

**모델**: GPT-5.1 (High Reasoning Capability) + RAG

**입력 (Multi-modal):**

- target_img: 등록상표 이미지 (Visual Check)
- vis_desc_text: Agent A가 생성한 외관 묘사 텍스트 (Textual Reference)
- target_text & target_product: 상표명 및 지정상품
- rag_results: RAG를 통해 검색된 유사 거절 사례 3~5건
- sem_desc: 관념 모델(모델 4)의 중간 산출물인 상표 관념 묘사 텍스트

**Role (Cross-Check Logic):**<구체적 Role 이나 시스템 프롬프트는 가다듬을 필요 있음 아래 거는 일단 예시임>

- **Vision Check**: 이미지를 직접 보며 도형의 복잡도와 심미감을 1차 파악한다.
- **Text Check**: Agent A의 묘사를 읽으며 혹시 놓친 미세한 특징이 있는지 확인한다.
- **Legal Check**: RAG 판례를 기준점으로 삼아, 파악된 특징이 '식별력 인정' 사유인지 '거절' 사유인지 대입한다. (예: "RAG 사례에서 '맛있는'은 성질표시로 거절되었으니, 본건의 'Delicious'도 식별력이 약하다.")

**System Prompt Directive:**

- "당신은 특허청 심사관입니다. 제공된 이미지와 묘사 텍스트를 모두 참고하여 상표를 입체적으로 분석하십시오."
- "RAG로 제공된 판례는 구법일 수 있으니 논리 위주로 참고하여 신법(제33조) 논리로 해석하십시오."
- "이미지와 텍스트를 종합하여, 식별력 유무(Strong/Weak/None)를 판단하십시오."
- "결론(식별력 없음)을 내릴 경우, 반드시 '어떤 거절 논리(예: 성질표시, 관용표장)'에 해당하는지 명시하십시오."

**Output (Structured JSON):**

- **Visual_Distinctiveness**: [None / Weak / Strong]
  - 판단 기준: 흔한 도형(원, 네모)인가? 독창적인 도안인가? 즉 상품의 보통명칭, 기술적 표장등(33조 1항 각호 어느 케이스에 해당하는 식별력 없는 표장인가)

- **Phonetic_Distinctiveness**: [None / Weak / Strong]
  - 판단 기준: 텍스트의 소리(발음)가 상품의 성질을 직감시키는가? 즉 상품의 보통명칭, 기술적 표장등(33조 1항 각호 어느 케이스에 해당하는 식별력 없는 표장인가)

- **Semantic_Distinctiveness**: [None / Weak / Strong]
  - 판단 기준: 텍스트나 도형의 의미(뜻)가 상품의 품질/효능을 설명하는가? 즉 상품의 보통명칭, 기술적 표장등(33조 1항 각호 어느 케이스에 해당하는 식별력 없는 표장인가)

- **Dominant_Part**: [Visual / Phonetic / Semantic / Composite]
  - 판단 기준: 소비자의 주의를 끄는 가장 지배적인 부분은 무엇인가?(외관, 호칭, 관념 중) – <참고: 만일 llm이 이런 판단이 어려울 것 같다면 이 부분은 뺄 수도 있음>

### Phase 3: 가중치 결정 (Dynamic Weighting - CoT) – phase3는 phase2 등과 별도의 llm 필요?

Agent B의 판단 결과를 수치적 가중치 벡터 $(W_{vis}, W_{pho}, W_{sem})$로 변환한다.

#### Step 1: 가중치 구간(Tier) 매핑 (Rule-Based)

각 요소별 식별력 등급에 따라 기본 가중치 구간을 할당한다. (※ 구체적 값은 추후 튜닝 예정)

- **None (식별력 없음)**: 0.0 ~ 0.1 (사실상 무시)
- **Weak (약함)**: 0.2 ~ 0.4
- **Strong (강함)**: 0.5 ~ 0.7
- **Dominant (요부)**: 해당 영역에 +0.2 ~ +0.3 가산점 부여

#### Step 2: 세부 수치 확정 (LLM)

**Prompt:**
"현재 [호칭 식별력: Weak], [관념 식별력: None], [요부: Visual]로 판단되었습니다. 이에 따라 가중치 구간은 $W_{vis}(High), W_{pho}(Low), W_{sem}(Min)$입니다. 상품('의류')과의 구체적 관계를 고려하여 각 가중치를 0.0~1.0 사이의 소수점 둘째 자리까지 확정해 주십시오. (단, $W_{vis}+W_{pho}+W_{sem}$의 합이 1.0이 될 필요는 없음)"

**Output**: weights = { "visual": 0.8, "phonetic": 0.3, "semantic": 0.1 }

### Phase 4: 최종 침해 확률 산출 (Hybrid Scoring Logic)

단순한 가중 평균은 낮은 유사도 점수가 높은 위험 점수를 희석(Dilution)시키는 문제가 있다. 따라서 본 시스템은 상표법의 법리적 판단 기준에 따라 **[Case A: 요부 관찰]**과 **[Case B: 전체 관찰]**로 분기하여 최종 점수를 산출한다.

#### 1. 점수 정규화 및 전처리 (Pre-processing)

모든 기초 유사도 점수($S$)를 0.0 ~ 1.0 범위로 강제 고정(Clipping)하여 연산 오류를 방지한다.

$$S'_{vis} = \text{clamp}(S_{vis}, 0.0, 1.0)$$
$$S'_{pho} = \text{clamp}(S_{pho}/100, 0.0, 1.0)$$
$$S'_{sem} = \text{clamp}(S_{sem}, 0.0, 1.0)$$

#### 2. 채점 로직 분기 (Decision Branching)

각 요소(외관, 호칭, 관념) 중 **"유효한 가중치를 가지면서 임계점을 초과하는 항목"**이 있는지 확인한다.

**판단 기준 (Thresholds):**

- **유효 가중치 기준 ($T_w$)**: 0.3 (식별력이 미미한 부분은 제외)
- **유사도 임계값 ($T_s$)**: 0.85 (침해가 확실시되는 점수)

**Trigger Condition:**
$$\text{IF } (W_i \ge T_w \text{ AND } S'_i \ge T_s) \text{ for any } i \in \{vis, pho, sem\}$$
$$\rightarrow \text{[Case A: 요부 관찰 로직] 적용}$$
$$\text{ELSE}$$
$$\rightarrow \text{[Case B: 전체 관찰 로직] 적용}$$

#### 3. 최종 점수 산출 (Calculation)

##### [Case A] 요부 관찰 (Dominant Part Rule)

**법리**: 상표의 어느 한 부분이 유사하여 전체적으로 혼동을 일으킬 우려가 있는 경우, 나머지 부분이 다르더라도 유사상표로 본다.

**산출 방식**: 임계점을 넘은 항목 중 **최댓값(Max Score)**을 최종 점수로 채택한다. (평균에 의한 희석 방지)

$$Final Score = \max(S'_{vis}, S'_{pho}, S'_{sem}) \quad (\text{where } W_i \ge T_w)$$

##### [Case B] 전체 관찰 (Overall Observation Rule)

**법리**: 결정적으로 유사한 부분은 없으나, 외관·호칭·관념이 전반적으로 유사하여 전체적인 인상(Impression)이 혼동을 줄 우려가 있는 경우를 판단한다.

**산출 방식**: 가중 RMS (Weighted Root Mean Square)

단순 평균보다 높은 점수에 더 큰 가중치를 부여하여, "어설픈 비유사" 항목들이 "잠재적 위험" 항목을 덮어버리는 것을 방지한다.

$$Final Score = \sqrt{ \frac{W_{vis} \cdot (S'_{vis})^2 + W_{pho} \cdot (S'_{pho})^2 + W_{sem} \cdot (S'_{sem})^2}{W_{vis} + W_{pho} + W_{sem}} }$$

#### 4. 위험도 등급 판정 (Risk Classification)

최종 산출된 $Final Score$에 따라 위험군을 분류한다. (※ 구간은 추후 조정)

- **0.90 ~ 1.00**: [고위험 (High Risk)] - 침해 확실 (Case A가 주로 해당됨)
- **0.75 ~ 0.89**: [중위험 (Medium Risk)] - 침해 가능성 높음 (경고 대상)
- **0.55 ~ 0.74**: [저위험 (Low Risk)] - 유사성 존재하나 항변 가능 (주시 대상)
- **0.00 ~ 0.54**: [안전 (Safe)] – 비유사

## 6. 미해결 혹은 고민 사항

### 등록 상표 vs 타인 사용 상표의 식별력 충돌

등록 상표(Target)에서는 식별력이 없다고 판단되는 부분이, 타인 사용 상표(Candidate)에서는 디자인적 요소로 인해 식별력이 인정될 여지가 있는 경우(혹은 그 반대) 처리가 복잡함.

**현재의 결론**: 프로젝트 목적상 '등록상표(Target)'를 기준으로 식별력 유무와 가중치를 판단하는 것이 타당함. 등록상표의 권리 범위가 어디까지 미치는지가 핵심이기 때문임.

### 모델2,3,4 의 점수 평탄화 문제

모델 2(Visual)와 모델 4(Semantic)는 코사인 유사도 기반이라 0.0 ~ 1.0이 나오지만, 보통 0.7만 넘어도 매우 높은 점수. 반면 모델 3(Phonetic)의 Fuzz Ratio 기반 점수는 80점 정도는 나와야 비슷하다고 봄.서로 다른 척도(Scale)와 분포를 가진 점수들을 그대로 섞으면(Ensemble), 특정 모델이 결과를 독식하거나 왜곡할 수 있음. 이를 해결하기 위한 [점수 정규화 (Normalization) 전략]이 필요. 이부분은 아직 고민중. 아래 절취선 이후로는 llm의 답변. 일반 이부분의 문제는 나중에 모델2,3,4 완성본 받으면 해결하고 지금은 이부분은 놔둔채 모델5의 다른 부분부터 개발시작(프로젝트 기간 맞추기위해.)

## 🚀 전략: "앵커 포인트(Anchor Point) 기반 구간 매핑"

복잡한 시그모이드(Sigmoid) 함수를 학습시킬 시간이 없으므로, 각 모델마다 **"안전(Safe) - 경계(Caution) - 위험(Danger)"**의 기준점을 잡고, 이를 통일된 기준(0~1)으로 강제 매핑하는 '구간별 선형 변환(Piecewise Linear Transformation)' 방식을 추천합니다.

### 1. 각 모델의 '체감 온도' 맞추기 (기준점 설정)

우선 각 모델 개발자(또는 테스트 결과)를 통해 아래 3가지 포인트를 정의해야 합니다.

| 모델          | 점수 범위 | 0점 (완전 다름) | 0.5점 (애매함/경계) | 1.0점 (똑같음) | 비고                                                           |
| ------------- | --------- | --------------- | ------------------- | -------------- | -------------------------------------------------------------- |
| 모델 2 (외관) | 0.0 ~ 1.0 | 0.0 ~ 0.4       | 0.5 ~ 0.6           | 0.7 이상       | Triplet Loss 특성상 0.7만 넘어도 매우 유사할 수 있음           |
| 모델 3 (호칭) | 0 ~ 100   | 0 ~ 40          | 50 ~ 65             | 70 이상        | 70점 이상이면 침해로 간주(Threshold)                           |
| 모델 4 (관념) | 0.0 ~ 1.0 | 0.0 ~ 0.6       | 0.6 ~ 0.8           | 0.85 이상      | 임베딩 특성상 기본 유사도가 높게 나옴 (0.6도 비유사일 수 있음) |

### 2. 통일된 기준(Unified Score)으로 변환

이제 위 표의 분포를 [0.0 ~ 1.0]의 정규화된 위험도로 쫙 펴주는 함수를 만듭니다.

입력이 무엇이든 0.8 이상이면 "위험", **0.5 정도면 "검토 필요"**가 되도록 맞추는 것입니다.

### 🛠️ 구현 가이드 (Python)

numpy.interp (선형 보간)를 사용하면 아주 쉽게 구현할 수 있습니다. 이 코드를 모델 5의 전처리 단계(Pre-processing)에 넣으십시오.

```python
import numpy as np

def calibrate_score(raw_score, model_type):
    """
    서로 다른 분포를 가진 점수들을 통일된 위험도(0~1)로 보정(Calibration)합니다.
    """
    if model_type == 'visual': # 모델 2
        # 입력 범위: [확실한 비유사, 애매함, 확실한 유사]
        # 외관은 0.7만 넘어도 꽤 유사하다고 판단 (가정)
        xp = [0.0, 0.4, 0.6, 0.8, 1.0]  # 원래 점수 구간
        fp = [0.0, 0.2, 0.5, 0.9, 1.0]  # 변환될 위험도 (Target)

    elif model_type == 'phonetic': # 모델 3
        # 호칭은 50점 밑은 무의미, 70점부터 급격히 위험해짐
        xp = [0, 40, 60, 70, 80, 100]
        fp = [0.0, 0.1, 0.5, 0.8, 0.95, 1.0]

    elif model_type == 'semantic': # 모델 4
        # 임베딩은 기본적으로 0.6 정도가 나옴. 0.85는 되어야 의미 있음.
        xp = [0.0, 0.6, 0.75, 0.85, 1.0]
        fp = [0.0, 0.0, 0.4, 0.9, 1.0]

    else:
        return raw_score

    # 선형 보간으로 점수 변환
    calibrated_score = np.interp(raw_score, xp, fp)

    # 0~1 사이로 확실하게 클리핑
    return float(np.clip(calibrated_score, 0.0, 1.0))

# --- 사용 예시 ---
# 모델 4에서 0.78이 나옴 -> 수치상으론 높지만, 관념적으론 애매한 점수
print(f"관념(0.78) -> 보정 점수: {calibrate_score(0.78, 'semantic'):.2f}")
# 결과: 0.5 (위험도 중간으로 조정됨)

# 모델 3에서 75점이 나옴 -> 수치상 0.75지만, 실제론 매우 위험한 점수
print(f"호칭(75) -> 보정 점수: {calibrate_score(75, 'phonetic'):.2f}")
# 결과: 0.88 (고위험으로 상향 조정됨)
```

### 💡 이 방식의 장점 (Why this works for MVP)

- **독식 방지**: 모델 4가 항상 0.8점씩 나온다고 해서 결과가 왜곡되지 않습니다. (0.8을 0.5로 눌러주니까요.)
- **직관적 튜닝**: "어? 호칭 모델이 너무 엄격한데?" 싶으면 xp (입력 구간) 리스트의 숫자만 살짝 바꾸면 됩니다. 모델을 재학습할 필요가 없습니다.
- **설명 가능성**: 심사위원에게 "각 모델의 특성에 맞춰 위험도 분포를 정렬(Alignment)했습니다" 라고 말하면 엔지니어링 역량을 어필하기 좋습니다.

### 결론:

당신의 직감이 정확했습니다. 그냥 섞으면 안 됩니다.

모델 5 가중치 연산에 들어가기 직전에, 위와 같은 Calibration 함수를 통과시켜서 모든 점수를 '실질적 위험도'로 변환한 후에 가중치를 곱하고 Max/RMS 연산을 수행하십시오.

### 전략 2. "기준점 잡는 법" (직관적 Heuristics)

나중에 튜닝할 때도, 판례를 다 뒤져서 통계를 낼 시간은 없습니다. **엔지니어적 직관(Heuristics)**으로 잡으시면 됩니다. 제가 **가이드라인(Cheat Sheet)**을 드릴 테니, 나중에 이 값을 초기값(Initial Value)으로 박으십시오.

#### ① 모델 3 (호칭): 이미 답이 나와 있습니다.

당신이 공유해준 scorer.py 코드를 보면 이미 기준이 있습니다.

- 80점 이상 -> High
- 50점 이상 -> Medium
- 나머지 -> Low

이게 곧 기준점입니다.

- Safe (0.0): 0점 ~ 40점
- Caution (0.5): 50점 ~ 60점
- Danger (1.0): 80점 이상

#### ② 모델 2 & 4 (코사인 유사도): "0.5의 함정"만 피하면 됩니다.

코사인 유사도(Cosine Similarity)는 벡터 공간의 특성상 0점(직각)이 나오기가 힘듭니다. 대부분 0.3~0.4는 기본으로 깔고 갑니다.

- Safe (0.0): 0.0 ~ 0.5 (0.5 밑으로는 사실상 남남입니다.)
- Caution (0.5): 0.6 ~ 0.75 (뭔가 분위기는 비슷한데 딱 꼬집기 힘든 구간)
- Danger (1.0): 0.85 이상 (거의 빼박)
