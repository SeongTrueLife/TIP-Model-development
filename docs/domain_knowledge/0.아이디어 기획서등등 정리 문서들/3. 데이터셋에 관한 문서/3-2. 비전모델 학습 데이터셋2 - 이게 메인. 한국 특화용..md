# AI Hub 기반 한국 상표/로고 데이터셋 활용 가이드

본 문서는 T-Guardian의 Vision 모델 및 한글 인식 모델 개발을 위해 **AI Hub(aihub.or.kr)**에서 즉시 활용 가능한 데이터셋 목록과 구체적인 활용 전략을 정리한 보고서입니다.

---

## 1. 핵심 발견: "원하던 데이터가 AI Hub에 있습니다"

사용자님의 직관이 정확했습니다. AI Hub에는 T-Guardian 개발에 필수적인 **"한글 OCR(간판)", "상표 이미지", "위조 상품"** 관련 데이터셋이 존재합니다. 
특히 **"유사 상표 이미지 검색 서비스의 사용자 입력 이미지 데이터"**는 우리의 목적과 90% 이상 일치하는 핵심 자산입니다.

### 🚀 활용 가치 요약

| 카테고리 | 데이터셋 명칭 | 활용 용도 | 중요도 |
| :--- | :--- | :--- | :--- |
| **상표 탐지** | **유사 상표 이미지 검색 사용자 데이터** | 상표 객체 탐지 및 유사도 비교 학습 | ⭐⭐⭐⭐⭐ (핵심) |
| **한글 OCR** | **야외 실제 촬영 한글 이미지** | 간판, 현수막 등 "현장 한글" 인식 | ⭐⭐⭐⭐⭐ (필수) |
| **디자인** | **스케치/아이콘 인식 추상 이미지** | 로고의 도안화/추상화 패턴 학습 | ⭐⭐⭐⭐ |
| **위조품** | **상품 이미지 데이터 (마크비전)** | 실제 유통되는 상품의 진품/가품 학습 | ⭐⭐⭐⭐ |

## 2. 데이터셋 상세 분석 및 활용 레시피

### 2.1 [핵심] 유사 상표 이미지 검색 서비스 사용자 입력 데이터
*   **구축 기관:** 마크클라우드 컨소시엄 (상표 검색 전문 기업)
*   **데이터 규모:** 약 27만 건 (원천), 51만 건 (라벨링)
*   **특징:**
    *   실제 사용자가 검색을 위해 업로드한 이미지이므로, **"저화질, 기울어짐, 조명 반사"** 등 우리가 찾던 **'현장 노이즈'**가 포함되어 있습니다.
    *   **YOLOv8** 학습용 라벨링(Bounding Box)이 되어 있어 즉시 투입 가능합니다.
    *   비엔나 코드(도형 분류 코드)가 태깅되어 있어, "도형 상표" 분류 학습에 최적입니다.
*   **T-Guardian 활용법:**
    *   이 데이터를 Vision 모델의 **Main Dataset**으로 사용합니다.
    *   해외 데이터(LogoDet-3K)보다 우선순위를 높여, "한국형 상표 탐지 모델"의 베이스라인으로 삼습니다.

### 2.2 [OCR] 야외 실제 촬영 한글 이미지 (Text in the Wild)
*   **데이터 규모:** 수십만 장 이상의 간판, 현수막, 도로 표지판 이미지
*   **특징:**
    *   "궁서체", "캘리그라피", "손글씨" 등 **다양한 폰트의 간판 글자**가 포함됨.
    *   기존 OCR(Tesseract 등)이 가장 취약한 '세로 간판', '돌출 간판' 데이터가 풍부함.
*   **T-Guardian 활용법:**
    *   한글 상표명("스타벅스", "김밥천국")을 정확히 읽어내는 **한글 특화 OCR 엔진 튜닝**에 사용합니다.
    *   '파' vs '과' 같은 한글 OCR 오류를 획기적으로 줄일 수 있습니다.

### 2.3 [위조품] 상품 이미지 데이터 (위조 상품 모니터링)
*   **구축 기관:** 셀렉트스타 (with 마크비전)
*   **데이터 내용:** 가방, 신발, 의류 등 5개 카테고리의 상품 이미지
*   **특징:**
    *   온라인몰 위조 상품 모니터링을 위해 구축된 데이터.
    *   **진품 vs 가품**의 미세한 차이(박음질, 로고 위치 등)를 학습할 수 있음.
*   **T-Guardian 활용법:**
    *   "간판/매장" 뿐만 아니라, 추후 **"온라인 쇼핑몰 크롤링"** 기능 확장 시 핵심 데이터로 사용합니다.

### 2.4 [도형] 스케치, 아이콘 인식용 추상 이미지
*   **데이터 내용:** 50만 건의 추상 이미지 (로고 디자인과 유사한 형태)
*   **T-Guardian 활용법:**
    *   **"도형 상표"**의 유사도 판단력을 높이는 데 씁니다.
    *   사진(Real)과 스케치(Abstract) 간의 매칭 학습을 통해, "손으로 그린 짝퉁 로고"도 잡아낼 수 있습니다.

## 3. 수정된 데이터 확보 전략 (AI Hub 중심)

기존의 "해외 데이터 + 합성" 전략을 **"AI Hub(한국형 데이터) + 해외 데이터"** 전략으로 전면 수정합니다.

### Phase 1: AI Hub 데이터 다운로드 및 정제 (Week 1-2)
1.  **AI Hub 회원가입 및 사용 신청:** 기업/연구자 계정으로 데이터 반출 신청.
2.  **데이터 필터링:**
    *   `유사 상표 데이터`: 전체 다운로드 후 로고(도형) 위주 선별.
    *   `야외 한글 이미지`: '간판' 카테고리만 추출.

### Phase 2: 모델 학습 (Week 3-6)
1.  **Detection (탐지):** `유사 상표 데이터` + `LogoDet-3K` 혼합 학습. (한국 상표 탐지율 95% 목표)
2.  **Recognition (인식):** `야외 한글 이미지`로 OCR 엔진 Fine-tuning. (간판 인식률 90% 목표)
3.  **Metric (비교):** `위조 상품 데이터`를 활용해 진품/가품 미세 차이 학습.

### Phase 3: 검증 (Week 7~)
*   실제 거리에 나가서 스마트폰으로 간판을 찍어보고, AI가 정확히 인식하는지 테스트.
