PART 1

상표법 기초

개발자가 알아야 할 최소한의 법리

1. 권리와 감시의 분리

등록 (특허청)

국가가 "독점권"을 부여합니다.

심사관의 영역

감시 (시장)

도둑을 잡는 건 주인의 몫입니다.

국가는 감시해주지 않습니다.

AI의 역할

변리사를 대신해 전국 시장을 감시하는 눈이 됩니다.

2. 침해 판단 기준: 오인·혼동

"소비자가 헷갈리면 침해다"

  • 1. 외관 (Visual):
    생김새, 디자인, 로고 형상
  • 2. 호칭 (Sound):
    부르는 소리, 발음
  • 3. 관념 (Meaning):
    뜻, 의미, 번역
상표 유사판단 3요소

사례 1: 외관(Visual) 침해

진품(좌) vs 유사상표(우)

글자가 달라도 침해입니다.

오른쪽 로고의 글씨가 다르더라도,


✅ 초록색 원형 테두리

✅ 중앙의 배치 구도


이것만으로 소비자는 "같은 브랜드인가?"라고 착각합니다.

→ OCR(글자 인식)만으로는 잡을 수 없습니다.

사례 2: 호칭(Sound) 침해

Case A

PUMA vs PUMMYS

앞부분 'Pu-' 발음이 강하게 유사하여 청각적 혼동 유발.

→ 침해 (유사)

Case B

LION vs RYAN

발음은 비슷할 수 있으나,

'사자' vs '사람 이름'으로 관념이 다름.

→ 비유사 (안전)

단순 텍스트 매칭이 아닌, 발음 기호 변환(G2P)이 필요합니다.

사례 3: 관념(Meaning) 침해

번역하면 같은 뜻?

Case:
"CROWN" (영어) vs "왕관" (한글)

Result:
외관/호칭은 전혀 다르지만,
관념(의미)이 동일하여 침해입니다.
Crown = 왕관

3. 지정상품 (Category)의 중요성

애플(컴퓨터) vs 애플(과일)

이름이 같다고 다 내 거 아님

상표권은 등록한 '카테고리' 안에서만 유효합니다.

IF (Mark="Apple") AND (Goods="Laptop")
>> INFIRNGEMENT (침해)

IF (Mark="Apple") AND (Goods="Fruit")
>> SAFE (비유사)

※ T-Guardian은 니스(Nice) 분류 코드를 자동 매핑합니다.

Track 1: Vision (YOLOv8)

로고만 찾아냅니다.

거리 풍경 사진에서 간판의 '상표 부분'만 정확히 떼어냅니다(Crop).


비교합니다. (Siamese Net)

등록 상표 이미지와 1:1로 비교하여 유사도 점수(0~100%)를 계산합니다.

Similarity: 87.5%
객체 탐지 및 비교

Vision 학습 데이터 전략

"진짜 침해 이미지는 구하기 어렵다"

특허청 데이터는 깨끗한 원본뿐입니다. 우리는 '현장 데이터'가 필요합니다.


해결책: AI Hub + 합성(Synthetic)

1. AI Hub: '유사 상표 이미지' 데이터셋 활용

2. 합성: 깨끗한 로고에 노이즈/기울기/조명을 적용해 가짜 현장 데이터 생성

Data Augmentation (데이터 증강)

Track 2 & 3: 언어적 판단

호칭 (G2P 알고리즘)

한국어 발음 변환 규칙 적용

"신라" → [실라]
"종로" → [종노]

소리나는 대로 변환 후 비교

관념 (Embedding)

단어를 벡터 공간에 매핑

Vec("Lion") ≈ Vec("사자")
Distance = 0.02 (매우 가까움)

Final Output: RAG Legal Report
