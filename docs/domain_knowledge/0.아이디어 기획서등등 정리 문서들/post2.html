09. 3-Track AI 분석 엔진

한국 상표법의 판단 기준(호칭, 외관, 관념)을 독립적인 모델로 구현했습니다.

Track 1: 호칭

유사도 85%

유사도 40%

Track 3: 관념

Conceptual Similarity

"Lion" vs "사자"
유사도 98%

10. Track 1: 호칭 유사도 (The Phonetic)

핵심 기술: G2P + 알고리즘

  • 문제: 딥러닝은 과유불급. 한국어 음운 규칙은 명확함.
  • 해결: KoG2P (Grapheme-to-Phoneme) 변환
  • 알고리즘: Levenshtein Distance (편집 거리)
# 호칭 분석 로직 예시
text_a = "국물" -> G2P -> "궁물"
text_b = "궁물" -> G2P -> "궁물"

distance = levenshtein("궁물", "궁물")
similarity = 1 - (distance / max_len)
# Result: 100% Match

11. Track 2: 외관 유사도 (The Visual)

핵심 기술: Metric Learning

  • 문제: 단순 분류(Classification)가 아님. "얼마나 비슷한가?"를 측정해야 함.
  • 해결: Siamese Network (샴 네트워크)
  • Loss Function: Triplet Loss (Anchor, Positive, Negative)
  • 데이터: AI Hub 한국 상표 데이터 활용

12. Track 3: 관념 유사도 (The Conceptual)

핵심 기술: LLM Embedding

  • 문제: "Apple"과 "사과"는 글자가 다르지만 뜻은 같음.
  • 해결: 거대 언어 모델(LLM)의 의미 공간(Semantic Space) 활용
  • API: OpenAI text-embedding-3-small 또는 Solar
# 관념 분석 로직
vec_a = openai.Embedding.create("Apple")
vec_b = openai.Embedding.create("사과")

score = cosine_similarity(vec_a, vec_b)
# Result: 0.92 (High Similarity)

13. Decision Engine: 침해 판정 로직

"어느 하나만 유사해도 오인·혼동을 일으키면 침해입니다." (대법원 판례)

Weighted Max-Pooling Strategy

final_score = max(
    w_p * phonetic_score,  # 호칭 (가중치 0.4)
    w_v * visual_score,    # 외관 (가중치 0.35)
    w_c * conceptual_score # 관념 (가중치 0.25)
)

if final_score >= 80:
    return "HIGH_RISK" (🔴)
elif final_score >= 50:
    return "MEDIUM_RISK" (🟡)
else:
    return "LOW_RISK" (🟢)
