# 상표권 침해 감지 시스템 - 기능 중심 정리
## 사용자 관점에서 본 시스템의 전체 기능

---

## 🎯 시스템의 핵심 가치

**변리사가 하는 일:**
```
기존 (수동)                          당신의 시스템 (자동)
├─ 고객 상표 수동 모니터링     →    ├─ 자동으로 계속 모니터링
├─ SNS, 지도 하나하나 확인     →    ├─ 모든 채널 동시 감시
├─ 침해 여부 판단 (변리사 경험) →    ├─ AI가 침해 가능성 제시
├─ 침해 판정 후 증거 수집     →    ├─ 자동으로 증거 제시
└─ 수동으로 권리행사 준비     →    └─ 템플릿 기반 자동 생성
```

**당신의 시스템이 변리사에게 주는 가치:**
- ⏰ **시간 절약**: 수동 모니터링 → 자동 모니터링 (시간 90% 감소)
- 🎯 **정확도 향상**: AI 기반 판단 + 변리사 최종 검증
- 📊 **데이터 기반**: 직관이 아닌 점수 기반 의사결정
- 🚀 **빠른 대응**: 침해 발견 → 권리행사까지 단축

```

│  ├─ 등록번호, 등록일자 입력

├─ 상표 수정 / 삭제

│  └─ 모니터링 중지 / 재개
│
└─ 상표 그룹화
   ├─ "같은 회사의 상표들" 그룹화
   ├─ "유사한 상표들" 자동 그룹화

### 2️⃣ 자동 크롤링 & 데이터 수집 (Automatic Crawling & Data Collection)

**사용자:** 시스템 (자동)

**기능:**
```
┌─ 네이버 지도 크롤링
│  ├─ 매일 자정에 자동 실행
│  ├─ 등록된 모든 상표명으로 검색
│  ├─ 검색 결과: 상점명, 주소, 전화, 이미지
│  ├─ 사업자 정보 추출
│  └─ 신규 발견 상점만 저장
│
├─ 카카오맵 크롤링
│  ├─ 네이버 지도와 동일
│  └─ 서로 다른 정보 보완
│
├─ 쇼핑몰 크롤링 (쿠팡, 무신사 등)
│  ├─ 상품명 검색
│  ├─ 판매자 정보, 상품 설명
│  └─ 상품 이미지
│
└─ SNS 모니터링 (선택사항)
   ├─ 인스타그램 해시태그 감시
   ├─ 페이스북 광고 감시
   └─ 신규 게시물 수집
```

**흐름:**
```
[크롤링 시작] 
    ↓
모든 등록된 상표 조회
    ↓
각 상표마다 "네이버 지도" 검색 (상품명 + 지역)
    ↓
검색 결과 수집 (상점명, 주소, 이미지, 연락처)
    ↓
데이터 정규화 & 중복 제거
    ↓
DB에 저장
    ↓
[크롤링 완료]
```

```

├─ 네이버 지도: 23개 발견 (신규 5개)
├─ 카카오맵: 18개 발견 (신규 2개)

### 3️⃣ 데이터 정규화 & 전처리 (Data Normalization & Preprocessing)

**사용자:** 시스템 (자동)

**기능:**
```
┌─ 텍스트 정규화
│  ├─ "헬 프 미" → "헬프미"
│  ├─ "HELPME" → "helpme"
│  ├─ "헬프-미" → "헬프미"
│  └─ "헬프미코리아" → 주요 부분 추출
│
├─ 중복 제거
│  ├─ 정확한 중복 제거
│  ├─ 유사 중복 감지 (같은 가게인데 여러 번 수집)
│  └─ 기록: "이 상점은 이미 알고 있던 것"
│
├─ 이미지 전처리
│  ├─ 이미지 크기 정규화 (모두 같은 크기로)
│  ├─ 노이즈 제거
│  ├─ 회전 보정
│  └─ OCR (텍스트 추출)
│
└─ 품질 검증
   ├─ 필수 필드 확인
   ├─ 유효성 검사 (전화번호 형식 등)
   ├─ 이상치 탐지
   └─ 품질 점수 부여 (100 ~ 50)
```

**예시:**
```
원본 데이터:
{
  "store_name": "헬 프 미 코 리 아",
  "address": "서울시 강남구 도산대로",
  "phone": "02-123-4567",
  "image_url": "https://..."
}

정규화 후:
{
  "store_name": "헬프미",
  "address": "서울시 강남구 도산대로",
  "phone": "02-123-4567",
  "image_url": "https://...",
  "normalized_name": "helpme",
  "quality_score": 95
}
```

### 4️⃣ AI 유사도 판단 (AI Similarity Judgment)

**사용자:** 시스템 (자동)

**기능:**

#### 4-1. 텍스트 기반 유사도 (외관, 호칭)

```
┌─ 외관 유사도 (Appearance Similarity)
│  입력: "헬프미" vs "헬프"
│  ├─ 문자 유사도 (Levenshtein): 0.80
│  ├─ N-gram 유사도: 0.85
│  ├─ 초성 유사도: 1.00 ("ㅎㅍㅁ" vs "ㅎㅍ")
│  └─ 최종 외관 점수: 85점
│
└─ 호칭 유사도 (Pronunciation Similarity)
   입력: "헬프미" vs "헬프"
   ├─ 초성 기반 ("헬" = "ㅎㅣ", "프" = "ㅍㅡ", "미" = "ㅁㅣ")
   ├─ 음성 변환 후 거리 계산
   ├─ 발음상 유사도: 0.92
   └─ 최종 호칭 점수: 92점
```

**화면 상에 보이는 것:**
```
[유사도 상세 분석]

텍스트 비교: "헬프미" vs "헬프"
────────────────────────────

📊 외관 유사도: 85점
   ├─ 문자 일치도: 80% (헬프 = 동일, 미 = 추가)
   ├─ 초성 일치도: 100% (ㅎㅍ 동일)
   └─ 해석: "매우 유사함"

📊 호칭 유사도: 92점
   ├─ 초성 기반: "ㅎㅍㅁ" vs "ㅎㅍ"
   ├─ 음성학적 거리: 매우 가까움
   └─ 해석: "매우 유사함 - 음상 발음상 거의 같음"
```

---

#### 4-2. 의미 기반 유사도 (관념)

```
┌─ 의미 유사도 (Concept Similarity)
│  입력: "헬프미" vs "도움이"
│  ├─ 단어 정의: "헬프미" = help + me
│  ├─ 단어 정의: "도움이" = 도움 + 이
│  ├─ KoBERT 임베딩 거리: 0.78 (높음 = 유사)
│  ├─ 의미 사전 확인: "help" ~ "도움" (동의어)
│  └─ 최종 관념 점수: 65점
│
└─ 다양한 경우들:
   ├─ "헬프미" vs "애플": 관념 5점 (전혀 다름)
   ├─ "헬프미" vs "헬프": 관념 50점 (약간 관련)
   └─ "헬프미" vs "도움이": 관념 65점 (상당히 유사)
```

**화면 상에 보이는 것:**
```
[의미 유사도 분석]

"헬프미" vs "도움이"
────────────────────

📊 관념 유사도: 65점

AI 분석:
├─ "헬프미" = "help" + "me" 
│  → 의미: "나를 도와주는 서비스"
│
├─ "도움이" = "도움" + "이"
│  → 의미: "도움을 주는 것"
│
└─ 의미적 거리: 중간 (상당히 유사)

근거:
└─ KoBERT 모델: 유사도 0.78 (높음)
```

---

#### 4-3. 이미지 기반 유사도 (로고)

```
┌─ 로고 이미지 유사도
│  입력: 로고1.png vs 로고2.png
│  ├─ 색상 분석: 유사도 0.75
│  ├─ 형태 분석 (CNN): 유사도 0.82
│  ├─ 구조적 유사도 (SSIM): 0.78
│  └─ 최종 이미지 점수: 78점
│
└─ 히트맵으로 표시
   ├─ 유사한 부분: 빨강 (매우 유사)
   ├─ 중간 유사: 노랑
   └─ 다른 부분: 파랑 (다름)
```

**화면 상에 보이는 것:**
```
[이미지 유사도 분석]

로고1 비교        로고2 비교        유사도 히트맵
[이미지]          [이미지]          [히트맵]

                  유사도: 78점
                  
📊 색상 유사도: 75점 (파란색 기조 동일)
📊 형태 유사도: 82점 (둥근 모양 동일)
📊 전체 구조: 78점 (로고 배치 유사)

결론: 이미지상 매우 유사함
```

### 5️⃣ 통합 침해 판단 (Integrated Infringement Judgment)

**사용자:** 시스템 (자동)

**기능:**
```
┌─ 점수 계산
│  입력: 
│  ├─ 외관: 85점
│  ├─ 호칭: 92점
│  ├─ 관념: 65점
│  ├─ 지정상품: 의류 vs 신발 (0.9 유사)
│  └─ 품질 신뢰도: 95점
│
├─ 가중치 적용
│  ├─ 외관: 85 × 0.35 = 29.75
│  ├─ 호칭: 92 × 0.35 = 32.20
│  ├─ 관념: 65 × 0.20 = 13.00
│  └─ 지정상품: 90 × 0.10 = 9.00
│
├─ 최종 점수
│  └─ (29.75 + 32.20 + 13.00 + 9.00) = 83.95점
│
└─ 위험도 판정
   ├─ 83.95점 ≥ 80점
   └─ → "침해 가능성 높음" ⚠️
```

**화면 상에 보이는 것:**
```
[침해 판정 결과]

┌─────────────────────────────┐
│  침해 위험도: 84점  ⚠️ HIGH  │
└─────────────────────────────┘

📊 점수 구성:
├─ 외관 유사도:  85점 ████████████████████ (35% 가중치)
├─ 호칭 유사도:  92점 ████████████████████ (35% 가중치)
├─ 관념 유사도:  65점 ███████████░░░░░░░░ (20% 가중치)
└─ 지정상품:    90점 ████████████████████ (10% 가중치)

결론: 침해 가능성이 높습니다.
권리행사를 적극 권장합니다.
```

## 🎯 기능 구현 우선순위 (MVP → Phase 2 → Phase 3)

### MVP (최소 기능, 2-3주)
```
필수 기능만 구현:
├─ 상표 등록 (간단함)
├─ 네이버 지도 크롤링 (공식 API)
├─ 텍스트 유사도 판단 (규칙 기반)
├─ 기본 대시보드 (리스트만)
└─ 간단한 알림 (이메일)
```

### Phase 2 (개선, 3-4주)
```
추가 기능:
├─ 의미 유사도 (KoBERT)
├─ 카카오맵 크롤링
├─ 상세정보 페이지
├─ 설명 가능성 (점수 분해)
└─ 사용자 피드백 수집
```

### Phase 3 (완성, 4-5주)
```
고급 기능:
├─ 이미지 유사도
├─ 자동 경고장 생성
├─ 월간 리포트
├─ 모델 재학습
└─ 시스템 설정
```

## 🏆 최종 정리

이 시스템의 핵심은:

**변리사가 해야 할 일:**
```
기존: 수동 크롤링 → 침해 판단 (경험에 의존) → 증거 수집 → 권리행사
현재: 자동 크롤링 → AI 제안 + 변리사 검증 → 자동화된 증거 → 권리행사
```

**사용자 여정:**
```
Week 1: 상표 3-5개 등록
Week 2-4: 시스템이 자동으로 침해 감지
Week 5: 변리사가 대시보드에서 결과 확인 후 권리행사
```

**변리사의 이득:**
```
✓ 시간 90% 절약
✓ 침해 감지율 95% 향상
✓ 대응 속도 10배 향상
✓ 데이터 기반 의사결정
✓ 자동화된 증거 수집
```
