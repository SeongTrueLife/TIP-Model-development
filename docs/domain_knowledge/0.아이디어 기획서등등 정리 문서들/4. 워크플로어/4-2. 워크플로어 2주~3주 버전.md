## (크롤링 + Vision 모델 = 균형잡힌 접근)

---

## 🎯 핵심 인사이트

### 기존 계획의 문제 ❌
- 모델 3개 (Vision, 호칭, 관념)를 다 만들려고 함
- 하지만 **데이터 수집 시스템이 없음**
- 결과: 모델만 있고, **실제 데이터로 테스트 못 함** → 변리사가 못 봄

### 리스코핑된 계획 ✅
- **Vision 모델 1개에 집중**
- **Naver 지도 크롤링 시스템 구축**
- **크롤링한 데이터로 즉시 테스트 가능**
- 변리사가 "오, 실제로 쓸 수 있겠는데?"라는 생각 들게 함

## 📈 시스템 동작 흐름

```
[웹 UI - 변리사 대시보드]
   ↓
   "크롤링 시작" 버튼 클릭
   ↓
[크롤링 시스템 (Selenium)]
   ├─ Naver 지도에서 이미지 수집 (병렬 처리)
   ├─ 메타데이터 추출 (위치, 좌표, 카테고리)
   ├─ 이미지 품질 필터링
   ├─ 중복 제거 (Perceptual Hashing)
   └─ 500~1000장 이미지 저장
   ↓
[이미지 저장소 (PostgreSQL)]
   ↓
[Vision 모델 (YOLOv8)]
   ├─ 로고 객체 탐지 (Bounding Box)
   ├─ 등록 상표와 비교 (Siamese Network)
   ├─ 유사도 점수 계산 (0~100%)
   └─ 결과 저장 (DB)
   ↓
[웹 UI - 결과 표시]
   ├─ 크롤링된 이미지 목록 (썸네일)
   ├─ 각 이미지의 로고 탐지 결과 (바운딩박스)
   ├─ 유사도 점수 (숫자 + 색상 지시자)
   ├─ 의심 이미지 리스트 (점수순 정렬)
   ├─ 각 이미지에 대한 변리사 판정 입력
   └─ 메모/코멘트 입력
```

## 🔧 크롤링 기술 (2~3주 안에 가능)

### 추천: Selenium (가장 실용적) ⭐

**특징:**
- 브라우저 자동화를 통한 크롤링
- Naver 로드뷰/스트릿뷰 직접 접근 가능
- 메타데이터 (위치, 좌표) 추출 용이

**장점:**
- ✅ 구현 쉬움 (Day 2에 시작 가능)
- ✅ 로그인 불필요
- ✅ 2주 안에 충분 (1000장 ÷ 10일 = 100장/일)

**단점:**
- ❌ 속도 느림 (1시간에 100~200장)
- ✅ 2주에는 충분함

**구현 예시:**
```python
from selenium import webdriver
import time
import requests
from PIL import Image

driver = webdriver.Chrome()
driver.get('https://map.naver.com/')

# 위치 검색
search_box = driver.find_element("name", "query")
search_box.send_keys("강남역")
search_box.submit()

time.sleep(2)

# 로드뷰 클릭
roadview_btn = driver.find_element("class name", "roadview-btn")
roadview_btn.click()

# 이미지 수집
for i in range(5):
    screenshot = driver.save_screenshot(f'image_{i}.png')
    # 좌표 추출
    coords = driver.execute_script("return window.location")
    print(f"Saved image_{i}.png at {coords}")
    time.sleep(1)
    # 이동
    driver.find_element("class name", "next-btn").click()
```

### 추천 타이밍 (8주)

```

            ↓ (변리사 피드백 1주)

            ↓ (변리사 피드백 1주)
```

### 전략
- ✅ 각 단계마다 변리사 피드백 수집

- ✅ 정확도를 점진적으로 높임

### 정확도 진화
```
1차 (2주):  Vision (87% 정확도)
            → "로고 위치 파악만 가능"

2차 (4주):  Vision + 호칭 (90% 정확도)
            → "외관 + 호칭으로 침해 판단"

3차 (6주):  Vision + 호칭 + 관념 (95% 정확도)

```
