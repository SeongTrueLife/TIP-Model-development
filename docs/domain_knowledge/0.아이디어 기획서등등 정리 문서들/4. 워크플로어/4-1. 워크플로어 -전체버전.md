## 1. 시스템 아키텍처 (High-Level Overview)

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       T-GUARDIAN SYSTEM ARCHITECTURE                    │
└─────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────────┐
│ [INPUT SOURCES] 데이터 수집                                               │
├────────────────────────────────────────────────────────────────────────────┤
│  ├─ 네이버 지도 크롤링 (로드뷰, 스트릿뷰)                                 │
│  ├─ 인스타그램 위치 태그 (사진)                                           │
│  ├─ 사용자 제보 (웹/모바일 앱)                                            │
│  └─ 카드사 가맹점 데이터 API (초기 필터링)                               │
└────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌────────────────────────────────────────────────────────────────────────────┐
│ [CRAWLER MODULE] 이미지 크롤링 및 전처리                                  │
├────────────────────────────────────────────────────────────────────────────┤
│  ├─ 이미지 다운로드 (병렬 처리)                                           │
│  ├─ 중복 제거 (Perceptual Hashing)                                       │
│  ├─ 낮은 품질 이미지 필터링                                               │
│  └─ 메타데이터 추출 (위치, 시간, 카테고리)                               │
└────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌────────────────────────────────────────────────────────────────────────────┐
│ [AI ANALYSIS MODULE] 3가지 AI 모델 분석                                   │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  ┌─ [1] VISION 모델 ────────────────────────────────────────────┐        │
│  │  입력: 이미지                                                 │        │
│  │  처리:                                                        │        │
│  │    1) YOLOv8 객체 탐지 → 로고 위치 찾기 (Bounding Box)      │        │
│  │    2) Siamese Network → 등록된 상표와 유사도 계산             │        │
│  │    3) 호칭/관념 모델 보조                                    │        │
│  │  출력: 로고 이미지 + 유사도 점수 (0~100%)                   │        │
│  │  신뢰도: 85~90%                                              │        │
│  └──────────────────────────────────────────────────────────────┘        │
│                                                                            │
│  ┌─ [2] 호칭 모델 ────────────────────────────────────────────┐         │
│  │  입력: OCR로 추출한 한글 텍스트                              │         │
│  │  처리:                                                        │         │
│  │    1) KoG2P로 음운 변환                                     │         │
│  │    2) Levenshtein Distance로 발음 거리 계산                │         │
│  │  출력: 발음 유사도 점수 (0~100%)                            │         │
│  │  신뢰도: 90~95%                                              │         │
│  └──────────────────────────────────────────────────────────────┘        │
│                                                                            │
│  ┌─ [3] 관념 모델 ────────────────────────────────────────────┐         │
│  │  입력: 상표명 텍스트                                         │         │
│  │  처리:                                                        │         │
│  │    1) OpenAI GPT-4 / Solar Embedding API 호출               │         │
│  │    2) 의미 벡터 변환                                         │         │
│  │    3) Cosine Similarity 계산                                │         │
│  │  출력: 의미 유사도 점수 (0~100%)                            │         │
│  │  신뢰도: 85~90%                                              │         │
│  └──────────────────────────────────────────────────────────────┘        │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌────────────────────────────────────────────────────────────────────────────┐

├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  입력: 3가지 모델의 점수                                                  │
│    - 외관 유사도 (V): 0~100%                                             │
│    - 호칭 유사도 (P): 0~100%                                             │
│    - 관념 유사도 (C): 0~100%                                             │

│  논리:                                                                    │
│    IF V > 80% THEN 외관상 의심 (⚠️)                                      │
│    IF P > 80% THEN 호칭상 의심 (⚠️)                                      │
│    IF C > 80% THEN 관념상 의심 (⚠️)                                      │

│    IF (V > 80% AND P > 80%) THEN 침해 가능성 높음 (🔴 HIGH)             │

│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌────────────────────────────────────────────────────────────────────────────┐
│ [REPORT GENERATION] LLM 기반 리포트 생성                                  │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  [JSON 데이터] (AI가 계산한 수치)                                        │
│    {                                                                      │
│      "target_trademark": "스타벅스",                                      │
│      "found_trademark": "스타박스",                                       │
│      "visual_similarity": 82,                                            │
│      "phonetic_similarity": 85,                                          │
│      "conceptual_similarity": 10,                                        │
│      "legal_assessment_criteria": "호칭 유사도 80% 초과"                 │
│      "risk_level": "HIGH"                                                │
│    }                                                                      │
│                         ↓                                                 │
│  [LLM 변환] (프롬프트 제한 방식)                                         │
│    System Prompt:                                                        │
│    "당신은 지식재산권 전문 보조원입니다. 아래 JSON 데이터만을 사용하여   │
│     문장을 완성하십시오. JSON에 없는 사실을 추가하거나 유추하지 마십시오."│

│  [최종 리포트] (변리사 검수용)                                           │
│    1. 호칭 분석:                                                        │
│       두 상표는 호칭 유사도가 85%로 매우 높습니다.                      │
│       '벅'과 '박'의 음가 차이는 있지만, 청감상 혼동 가능성이 큽니다.   │
│                                                                            │
│    2. 외관 분석:                                                        │
│       외관 유사도는 82%입니다. 전체적 형태와 색상이 유사합니다.        │
│                                                                            │
│    3. 관념 분석:                                                        │
```

## 4. 핵심 모듈 상세 설명

### 4.1 VISION 모듈 (외관 유사도)

**입력:** 이미지 파일
**처리:**
1. YOLOv8로 로고 객체 탐지
2. 탐지된 로고 추출
3. Siamese Network로 등록 상표와 비교
4. Cosine Similarity 계산

**출력:** 
```json
{
  "detected_logo": {"x": 100, "y": 200, "width": 50, "height": 50},
  "visual_similarity": 82,
  "confidence": 0.95
}
```

### 4.2 호칭 모듈 (청각적 유사도)

**입력:** 한글 텍스트 (상표명)
**처리:**
1. OCR로 간판에서 한글 추출
2. KoG2P로 음운 변환
3. Levenshtein Distance 계산

**출력:**
```json
{
  "target_name": "스타벅스",
  "found_name": "스타박스",
  "phonetic_similarity": 85,
  "edit_distance": 1
}
```

### 4.3 관념 모듈 (의미적 유사도)

**입력:** 상표명 텍스트
**처리:**
1. OpenAI GPT-4 Embedding API 호출
2. 벡터 간 Cosine Similarity 계산

**출력:**
```json
{
  "target_name": "스타벅스",
  "found_name": "스타박스",
  "conceptual_similarity": 10,
  "reason": "Starbucks=커피/음료, 스타박스=불명확"
}
```

### 4.4 Decision Engine (침해 판정)

**규칙:**
```
if (외관 > 80% AND 호칭 > 80%):
    risk_level = "HIGH" (🔴)
    reason = "외관과 호칭이 모두 유사"
    
elif (외관 > 75% OR 호칭 > 75%) AND 관념 > 70%:
    risk_level = "MEDIUM" (🟡)
    reason = "부분적 유사 + 관념도 유사"
    
else:
    risk_level = "LOW" (🟢)
    reason = "침해 가능성 낮음"
```

### 4.5 LLM 리포트 모듈

**프롬프트 전략:**
```
[System]
당신은 지식재산권 전문 보조원입니다. 
아래 [JSON 데이터]만을 사용하여 문장을 완성하십시오.
절대 JSON에 없는 사실을 추가하거나 유추하지 마십시오.

[JSON 데이터]
{
  "target": "스타벅스",
  "found": "스타박스",
  "visual": 82,
  "phonetic": 85,
  "conceptual": 10,
  "assessment": "호칭 유사도 80% 초과"
}

[출력 형식]
1. 호칭 분석: ...
2. 외관 분석: ...
3. 관념 분석: ...
4. 법적 평가: ...
[변리사 최종 검토 필요]
```

## 5. 변리사 대시보드 워크플로우

```
[변리사가 로그인]
    ↓
[침해 의심 상표 리스트 확인]
   (위험도별, 브랜드별, 지역별 정렬)
    ↓
[특정 건 선택]
    ↓
[상세 분석 리포트 조회]
   ├─ 원본 이미지 (지도 스크린샷 + 좌표)
   ├─ AI 분석 점수 (외관 82%, 호칭 85%, 관념 10%)
   ├─ 자동 생성 리포트 (LLM)
   ├─ 참고 법조 (대법원 판례 링크)
   └─ 추천 조치안 (경고장/소송/신청)
    ↓
[변리사 최종 판정]
   ├─ ✓ 침해 확정 → 조치 시작
   ├─ ? 추가 조사 → 재분석 신청
   └─ ✗ 비침해 → 종료
    ↓
[피드백 저장]
   (변리사의 판정 → 모델 학습에 활용)
```

## 6. MVP vs 최종 버전의 차이

### MVP (Week 12)
- ✅ Vision 모델: 기본 기능 (85~90%)
- ✅ 호칭 모델: KoG2P + Levenshtein
- ✅ 관념 모델: OpenAI API
- ✅ 간단한 Decision Logic
- ✅ 기본 대시보드 (수동 검수)
- ❌ 판례 자동 인용 (나중)
- ❌ 한글 특화 Fine-tuning (나중)

### 최종 버전 (6-12개월 후)
- ✅ Vision 모델: 한국 데이터 Fine-tuning (90%+)
- ✅ 호칭 모델: Siamese LSTM 고도화
- ✅ 관념 모델: 다국어 지원
- ✅ 고도화된 Decision Logic (법적 기준 반영)
- ✅ 고급 대시보드 (예측 순위, 트렌드 분석)
- ✅ 대법원 판례 자동 인용 (RAG)
- ✅ 변리사 협력 네트워크 (Feedback Loop)
